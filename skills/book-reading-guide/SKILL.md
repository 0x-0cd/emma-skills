# 书籍导读 Skill

当用户说「导读xxx」「xxx应该怎么读」「帮我把xxx导读一下」或类似表达时触发。

## 完整流程

### 第一步：搜集书籍内容

用 web_search 搜索这本书的大致内容，包括：
- 本书的核心主题和论述范围
- 目录结构概览
- 它在领域内的地位和影响力

然后用你自己的话**浓缩提炼**成一段类似论文摘要的概述（200-300字），不要照搬搜索结果。

### 第二步：分析前置知识

分析阅读这本书需要哪些前置知识，分两类：

**作者背景：**
- 作者生平、学术/职业背景
- 创作这本书时的时代背景和动机

**知识储备：**
- 需要哪些学科基础
- 需要熟悉哪些概念、术语、理论
- 如果是外文翻译书，是否需要了解原版文化背景

### 第三步：判断前置知识呈现方式（关键分支）

对第二步分析出的前置知识做分类处理：

**A. 简单的前置知识**（比如作者生平、时代背景、一个可以几句话解释清楚的概念）
→ 直接呈现给用户，不需要问

**B. 体系化的前置知识**（比如"没学过线性代数和概率论就看不懂这本机器学习"）
→ **先问用户对这个前置知识的掌握程度**，用 clarify 工具：
  - 如果用户说"懂/学过/没问题" → 继续导读
  - 如果用户说"不太懂/没学过" → **建议先学对应知识再来读这本书**，不要硬读
  - **关键：给出具体的学习路线图**——不要只说"你先去学一下XX"，而是给出：先学什么、再学什么、大概需要多久、用什么资源入门（推荐通俗入门资料而不是大部头教材）。让用户拿着这个路线图就能执行。

### 第四步：找最佳版本

对于中文原创书籍：找评价最高的版本（出版社、版次、豆瓣评分参考）
对于外文翻译书籍：找公认**翻译质量最高**的译本，对比不同译本的优缺点
对于外文原版书籍：找最新或公认最好的版本

用 web_search 搜索确认，优先参考豆瓣读书的评价。

### 第五步：输出结构化导读报告

最终呈现给用户的格式如下：

---

## 📖 《书名》导读

### 书籍概况
（200-300字的摘要式概述）

### 前置知识
**作者与背景：**
- ...

**必备知识：**
- ...

### 推荐阅读方式
（根据前置知识判断结果给出建议，比如可以直接读、建议先补XX知识再读、适合精读/泛读等）

### 推荐版本
- **书名 / 译者 / 出版社 / 版次**
- 推荐理由：...
- 备选版本：...

---

### 推荐阅读方式 的写法
根据第三步的判断结果，分两种情况：
- **用户可以直接读** → 给出精读/泛读建议、重点章节提示
- **用户需要先补前置知识** → 先陈述观点（建议先学什么再读），然后给出**具体的学习路线**（分步骤、带资源推荐、估算时间），最后说"等你准备好了再找我继续"

### 推荐版本 的搜索技巧
中文书籍的版本搜索有时会遇到困难（搜索引擎返回购物站而非书评）。尝试以下策略：
- 英文书搜 "book title best edition/translation review"
- 中文书搜 "书名 哪个版本好 豆瓣" 或 "书名 译本 对比"（注意：国内站点直连更快，不需要走代理）
- 经典著作（特别是马克思主义经典）中译本首选**人民出版社/中央编译局**版本
- 查不到具体评分或对比时可以基于出版社口碑和译者资历给出判断，但要说明这是基于一般规律的推荐而非实测对比

## 重要约束

1. **不需要**告诉用户在哪买、在哪下载
2. **不需要**附购买链接或下载链接
3. 如果搜不到某本书的信息，直接说没找到，不要编造
4. 推荐版本时优先说清楚理由，不只是抛个书名出去
